Manual de uso

Guia breve para operar TrafficWatch IDS dentro de una red autorizada.

1. Iniciar el dashboard

  1. Ejecuta abrir_cmd_proyecto.bat.
  2. Entra a http://127.0.0.1:5000.
  3. Mantén abierta la ventana de CMD o PowerShell mientras usas el sistema.

2. Usar desde otra PC

  1. Conecta ambas computadoras a la misma red.
  2. Usa la IP mostrada al iniciar el dashboard, por ejemplo http://192.168.1.X:5000.
  3. Abre /attack-lab desde la segunda PC para hacer pruebas controladas.

3. Interpretar alertas

  • ALTO: actividad que debe revisarse con prioridad.
  • MEDIO: patrón inusual que requiere contexto.
  • IP origen: equipo que generó el tráfico detectado.
  • Objetivo: equipo y puerto al que se dirigió el tráfico.

4. Probar el laboratorio

  1. Entra a Attack Lab desde un equipo autorizado.
  2. Elige una prueba de tráfico controlado o un escaneo Nmap autorizado.
  3. Regresa al Dashboard y revisa la alerta con IP, tipo, riesgo y descripción.

5. Bloquear una IP temporalmente

  1. Ejecuta el dashboard como administrador.
  2. Busca una alerta SSH: FUERZA_BRUTA_SSH o TRAFICO_REAL_LAB_FUERZA_BRUTA_SSH.
  3. Pulsa Bloquear 10 min en la columna Acción.
  4. Windows Firewall bloqueará conexiones entrantes de esa IP y quitará la regla al terminar el tiempo.

6. Revisar y exportar

  • Usa Historial para filtrar alertas anteriores.
  • Exporta resultados en JSON o CSV para informes.
  • Consulta Red local para identificar dispositivos detectados.
Solo realiza pruebas sobre equipos y redes autorizadas. El bloqueo temporal protege la computadora donde TrafficWatch está ejecutándose; no expulsa dispositivos del Wi-Fi ni bloquea todo el router.

Tipos de trafico

Referencia rapida para interpretar de donde viene y hacia donde va un paquete.

Contexto de red

El IDS detecta la IP local, el gateway y la red local. Con esos datos puede entender mejor si un paquete viene de internet, sale de la PC, pertenece a la red local o se comunica con el router.

Paquete entrante

Una IP externa envia trafico hacia tu computadora. Puede ser normal, pero si toca puertos sensibles o aparece muchas veces, merece atencion.

45.162.90.157 -> 192.168.1.33

Paquete saliente

Tu computadora envia trafico hacia internet. Suele pasar al navegar, actualizar programas o usar servicios en linea.

192.168.1.33 -> 8.8.8.8

Paquete local

Dos dispositivos de la misma red local se comunican entre si. Por ejemplo, otra PC, celular, impresora o servidor dentro de la LAN.

192.168.1.50 -> 192.168.1.33

Paquete hacia el gateway

Tu computadora se comunica con el router. Es normal cuando consulta servicios de la red o sale hacia internet mediante la puerta de enlace.

192.168.1.33 -> 192.168.1.1

Por que ayuda al IDS

Esta clasificacion reduce ruido y ayuda a interpretar mejor las alertas. No es lo mismo una conexion al puerto 22 desde internet hacia tu PC que una conexion desde tu PC hacia el router.

Reglas IDS

Resumen de reglas activas que generan alertas de seguridad.

ESCANEO_DE_PUERTOS

Detecta cuando una IP intenta acceder a varios puertos diferentes dentro de una ventana corta de tiempo.

Ejemplo: nmap -p 1-100 <gateway>

SYN_FLOOD

Detecta demasiados paquetes TCP SYN desde una misma IP en pocos segundos.

Riesgo: posible denegacion de servicio

ICMP_FLOOD

Detecta exceso de paquetes ICMP desde una misma IP, similar a un abuso de solicitudes ping.

Riesgo: trafico ICMP inusual

PUERTO_SOSPECHOSO

Genera alerta cuando se detecta conexion hacia puertos sensibles definidos en la configuracion.

Puertos: 21, 22, 23, 3389, 4444, 5900

FUERZA_BRUTA_SSH

Detecta intentos repetidos hacia el servicio SSH, normalmente asociado al puerto 22.

python simular_fuerza_bruta.py --port 22 --count 10

FUERZA_BRUTA_FTP

Detecta intentos repetidos hacia FTP, normalmente asociado al puerto 21.

python simular_fuerza_bruta.py --port 21 --count 10

FUERZA_BRUTA_TELNET

Detecta intentos repetidos hacia Telnet, servicio antiguo asociado al puerto 23.

python simular_fuerza_bruta.py --port 23 --count 10

FUERZA_BRUTA_RDP

Detecta intentos repetidos hacia Escritorio Remoto de Windows, normalmente asociado al puerto 3389.

python simular_fuerza_bruta.py --port 3389 --count 10

ALTA_FRECUENCIA_CONEXIONES

Detecta demasiadas conexiones TCP desde una misma IP dentro de pocos segundos, aunque no sean a muchos puertos diferentes.

python simular_fuerza_bruta.py --port 80 --count 120 --delay 0.01

PUERTO_RARO

Detecta conexiones hacia puertos poco comunes definidos en la configuracion, como 31337, 6667 o 9001.

nmap -p 31337 <gateway>
